# StoryWire デモ動画 台本

> **制約**: 4分以内 / デモパート約2:30 + ピッチパート約1:20
> **言語**: 日本語（ティッカー・音声ともに）

---

## 事前準備チェックリスト

```bash
# 1. デモ用ニュースをDBに投入（通常ニュースのみ、速報はまだ入れない）
python demo_setup.py --seed-db

# 2. M1の動画が content_index に登録されていることを確認

# 3. 画像アセット配置確認
#    images/trump_truth_kharg.png     ← Truth Social スクショ
#    images/iranstrikemap.png          ← OSINT中東マップ
#    images/kharg_island_map.png       ← Kharg Island地図

# 4. OBSのブラウザソース設定確認（http://localhost:8765/overlay）

# 5. gemini_live_client.py 起動（ティッカーサーバも自動起動）
#    または demo_setup.py --run --breaking-delay 0 でサーバのみ起動

# 6. OBS録画開始
```

---

## PART 1: デモパート（〜2:30）

### シーン1: オープニング + 通常放送（0:00 〜 0:25）

**画面構成**:
- MediaWindow: Iran Strike Map（中東マップ）を表示
- ティッカー: 通常ニュース7件がスクロール

**台本（ナレーション/字幕用）**:

> StoryWire は、ジャーナリストのライブ報道を AI がリアルタイムで支援するツールです。
> 現在、イラン紛争のライブ報道中です。
> 画面には中東の状況マップが表示され、下部のティッカーに最新ニュースが流れています。

**操作**:
- 起動前に `show_image iranstrikemap` でマップを表示しておく
- ティッカーは自動でスクロール中

---

### シーン2: Geminiに動画再生を指示（0:25 〜 0:55）

**操作**:
- F9（PTT）を押して音声指示

**ジャーナリスト（音声）**:
> 「ホルムズ海峡封鎖についての動画を見せて」

**Gemini応答**（音声）:
> 「ホルムズ海峡封鎖のレポートを再生します」

**画面**: M1生成の動画クリップ「ホルムズ海峡封鎖」が再生される（約10秒見せる）

**ジャーナリスト（音声）、動画再生中に**:
> 「次に、イランの新指導者についてのレポートを」

**Gemini応答**: 動画切替 → M1生成の「イラン新最高指導者」クリップへ

---

### シーン3: 速報差し込み（0:55 〜 1:25）

**操作**（別ターミナルまたはタイマー）:
```bash
python demo_setup.py --trigger-breaking
```

**画面変化**:
1. ティッカーのスクロール内に赤背景のアイテムが出現
   「**BREAKING** トランプ大統領、米軍がKharg Island（ハールク島）のイラン軍事施設を攻撃したと発表」
2. 上部バナーがフェードイン（8秒で消える）
3. 少し遅れてもう1つの速報も混入
   「**BREAKING** 原油価格WTI $100突破 — Kharg Island攻撃報道を受け急騰」

**ナレーション/字幕**:
> ニュースの収集パイプラインが速報を検知すると、自動的にティッカーに赤字で割り込みます。

---

### シーン4: ジャーナリストが速報を深掘り（1:25 〜 1:55）

**ジャーナリスト（音声）**:
> 「今の速報について詳しく教えて」

**Gemini応答**（音声）:
> 「トランプ大統領がTruth Socialで声明を投稿しました。
>  米中央軍がKharg Islandの全軍事目標を攻撃したとのことです。
>  石油施設は攻撃しなかったと主張していますが、
>  市場はイランの報復を警戒して原油が100ドルを超えています。
>  トランプ大統領の投稿を表示します。」

**画面**: `show_image trump_truth_kharg` → Truth Social投稿のスクショが表示される

---

### シーン5: 地図で場所を確認（1:55 〜 2:20）

**ジャーナリスト（音声）**:
> 「Kharg Islandの場所を地図で確認させて」

**Gemini応答**:
> 「Kharg Islandの地図を表示します。
>  ペルシャ湾のイラン沿岸にある島で、
>  イラン最大の原油輸出ターミナルがあります。」

**画面**: `show_image kharg_island_map` → Wikipedia地図が表示される

**ジャーナリスト（音声）**:
> 「ありがとう。全体のマップに戻して」

**Gemini応答**: `show_image iranstrikemap` → 中東マップに戻る

---

### シーン6: デモ締め（2:20 〜 2:30）

**画面**: 中東マップ + ティッカー（速報含む）がスクロール

**ナレーション**:
> このように、StoryWireはニュース収集からライブ放送支援まで、
> ジャーナリストのワークフローをAIがシームレスに支えます。

---

## PART 2: ピッチパート（2:30 〜 3:50）

**画面**: プレゼンスライドまたは画面収録の上にテキストオーバーレイ

### 解決する課題（2:30 〜 3:00）

> ライブ報道中のジャーナリストは、速報の確認、映像の切り替え、
> 地図やソースの表示など、膨大なマルチタスクに追われています。
>
> 従来は専門のディレクターやスイッチャーのチームが必要でした。
> 一人のジャーナリストがこれを全てこなすのは不可能に近い。

### StoryWireの価値（3:00 〜 3:30）

> StoryWire は Gemini をリアルタイム AI ディレクターとして活用し、
> この問題を解決します。
>
> 音声で指示するだけで、AIが最適な動画や地図を即座に表示。
> ニュースの自動収集・分析から動画クリップの生成まで、
> パイプライン全体をAIが担います。
>
> 速報が入れば自動でティッカーに表示され、
> ジャーナリストは報道に集中できます。

### マルチモーダル/エージェント技術（3:30 〜 3:50）

> 技術的には、Gemini Live API の音声入出力と Function Calling で
> リアルタイムの双方向対話を実現。
> ニュース収集パイプラインでは Gemini が記事の重要度を判定し、
> 速報フラグを自動付与。
> 動画クリップの生成には Gemini の画像生成と TTS を組み合わせています。
>
> StoryWire — AIが支える、次世代のジャーナリズム。

## 撮影コマンドまとめ

```bash
# ターミナル1: メインアプリ起動
python gemini_live_client.py

# ターミナル2: 速報差し込み（シーン3のタイミングで実行）
python demo_setup.py --trigger-breaking

# やり直し時: 速報フラグ解除
python demo_setup.py --clear-breaking
```
